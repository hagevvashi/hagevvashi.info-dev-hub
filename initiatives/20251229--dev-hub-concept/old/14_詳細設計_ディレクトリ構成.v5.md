# 4. 詳細設計_ディレクトリ構成

```text
dev-platform/
├── .devcontainer/
│   ├── devcontainer.json.template # ★変更: テンプレートとして管理
│   ├── Dockerfile
│   ├── host-setup.sh              # テンプレートからjsonを生成＆.env作成
│   └── post-create.sh             # コンテナ内セットアップ（clone等）
├── foundations/                       # ★確定した土台（Fixed情報）・・・チームのナレッジ
│   ├── onboarding/          # 新人向け手順
│   ├── operations/          # 運用マニュアル
│   ├── adr/                 # Architecture Decision Records
│   │   ├── 001_monolithic-devcontainer.md
│   │   └── 002_docker-volume-strategy.md
│   └── principles/          # 設計思想、コーディング規約
├── initiatives/                   # ★進行中の取り組み（流動的情報）
│   ├── user-authentication/
│   │   ├── _overview.md
│   │   ├── research/
│   │   ├── meetings/
│   │   ├── tasks/
│   │   └── decisions/
│   └── ...
├── members/                       # 個人の素振り場
├── workspace.code-workspace       # ★解説: マルチリポジトリへのショートカット定義
├── .gitignore                     # devcontainer.json, .env 等を除外
└── .env.example                   # 必要な環境変数の定義
```
